# 商店文案模板

当前优先用于 **Google Play** 提交；iOS 文案先保留，不作为当前阻塞项。

## 应用名称

一起选

## 一句话简介

给两个人一起决定今天吃什么的共享小助手。

## Google Play 短描述

双人共享待办、菜单、用餐记录和随机转盘，轻松决定今天吃什么。

## 长描述

“一起选”是一款适合情侣、室友或搭子一起使用的共享饮食小工具。

你们可以在同一个房间里记录今天的想法、安排待办、整理常吃菜单，还能在选择困难的时候直接用转盘帮忙做决定。

主要功能包括：

- 实时日历，按天查看记录
- 每日待办，安排今天谁来做什么
- 菜单管理，沉淀常吃的选项
- 用餐记录，方便快速回看
- 随机转盘，快速决定今天吃什么

如果你和另一台手机输入同一组房间码与配对口令，还可以共享同一份待办、菜单、用餐记录和转盘结果。

## iOS 预留副标题建议

双人共享菜单与吃饭决定工具

## iOS 预留关键词建议

吃饭,菜单,待办,情侣,转盘,日历,记录,共享

## Google Play 审核说明草案

本应用是一款双人共享饮食计划工具，本次提交的 Android 包包含 Supabase 云同步能力。

审核或测试时可通过以下方式验证核心功能：

1. 在单台设备上新增待办、菜单和用餐记录，确认本地可正常使用
2. 在两台设备上输入同一组房间码与配对口令，验证共享数据同步
3. 如只使用单台测试设备，也可完成本地记录与浏览，不影响基础功能验证

本次提交包的云同步能力依赖已配置的发布环境变量，不需要审核人员在 App 内额外配置服务参数。

1. 两个人的晚饭决定，一处搞定（首页上半屏总览）

## Data safety 口径草案

### 可按代码准备为“会处理”的数据
- 用户内容：待办、菜单自定义、用餐记录；如兼容历史版本数据，还可能包含历史留言内容
- 标识符：匿名会话与同步相关标识

### 可按代码准备为“未见使用”的能力
- 定位
- 相机 / 麦克风
- 通讯录
- 相册 / 文件读取
- 广告 SDK / 广告 ID
- 分析 SDK
- 支付 / 订阅

### 后台勾选草稿
- 收集到服务器：倾向填 是
- 共享给广告或第三方营销：倾向填 否
- 数据用途：至少包含 App 功能；涉及匿名会话、房间鉴权、恢复令牌、限流时，还要逐题核对是否涉及账号管理 / 安全 / 防滥用
- 传输中加密：确认生产环境为 `https://` 后再填 是
- 用户可请求删除数据：按你的实际后端与运营能力确认后再填

### 需要你在后台最终确认的项
- 是否勾选“收集到服务器”
- 是否勾选“传输中加密”
- 是否声明用户可请求删除数据
- 隐私政策是否已完整覆盖上述数据流

## 内容分级作答准备

可先按“生活工具 / 双人协作工具”思路准备：
- 不主打暴力、赌博、成人、药品、酒精、金融、新闻、医疗
- 当前版本不提供新的用户之间自由文本交流或公开社区能力
- 如后台问到用户交互、消息交流、用户生成内容，应按“当前版本不提供新的自由文本交流入口，但兼容历史版本数据时可能保留历史留言内容”逐题核对
